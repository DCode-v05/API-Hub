Quality layer

Generated clients retry & time out

Every converted TypeScript & Python SDK now retries transient failures (429, 502/503/504, dropped connections) with exponential backoff + jitter, honors Retry-After, and applies a 30s timeout per call — the production polish Stainless/Fern ship, on every language, from one place.

Versioning

Every conversion is stamped

Each converted artifact carries a deterministic cn.manifest.json: what it was converted from (source kind, title), the IR hash, a content version, and the operation count. Re-convert the same input → same version; a changed input → a new one. No timestamps or paths, so it stays reproducible.

How it works — three guarantees, one conversion

Nothing here touches GitHub push/pull. The three additions attach to the conversion itself: any source climbs to one IR, fans out to surfaces, and each surface is made production-quality, stamped, and tested.

Fig. — the three additions hang off the conversion, not a build/publish service.

The quality layer, under the hood. Each generated call wraps in a small loop — start a timeout, send; on a transient error wait min(2ⁿ, 8s) + jitter and retry (up to 2×), else surface the error. The generated bytes stay identical (only runtime behavior is smarter), so the determinism check still passes.

# baked into every converted SDK call
for attempt in 0..maxRetries:
  start 30s timeout
  res = send(request)
  if res is 429/5xx or network error: wait min(2^attempt, 8s)+jitter; retry
  else: return res          # success — or raise after the last attempt

Why it matters for a converter

Quality layer
A converted SDK that gives up on the first hiccup is a demo, not a deliverable. Retries + timeouts make the output something a team can actually ship — and because it lives in the generator, every language and direction gets it for free.
Versioning
Stainless/Fern version their SDKs so consumers know what changed. We get the same — a content-derived version + provenance in every artifact — without a hosted service: it's just a deterministic file the conversion writes.
Correctness QA
The hash test proves a conversion is reproducible; the QA suite proves it's correct (real operations, valid schemas, lossless round-trip). Together they replace per-language manual review with one command.

In one line: our conversions now match Stainless/Fern on quality, versioning, and correctness — delivered inside the conversion itself, with zero GitHub push/pull and zero hosted infrastructure.
